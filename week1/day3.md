# Day 3: REST APIs and HTTP Basics (1h)

- **Topics:** HTTP methods (GET, POST, PUT, DELETE) and RESTful API principles.  
- **Resources:** REST primer – [REST API tutorial](https://restfulapi.net/) (explains use of HTTP verbs); [Postman Blog: What is a REST API?](https://blog.postman.com/rest-api-examples/); MDN [Glossary: REST](https://developer.mozilla.org/docs/Glossary/REST). YouTube: *REST API Tutorial* (e.g. *freeCodeCamp* or *Traversy Media*).  
- **Exercise:** Use `curl` or Postman to make GET and POST requests to a public API, e.g. https://jsonplaceholder.typicode.com/posts. Confirm GET retrieves data (200 OK) and POST creates a new resource (201 Created).  
- **Why:** Understanding HTTP and REST is crucial for web APIs. RESTful APIs use standard HTTP verbs: GET to retrieve data, POST to create, etc. For example, GET does not change state, while POST creates new resources. A REST API is essentially an HTTP service that any client can call.
