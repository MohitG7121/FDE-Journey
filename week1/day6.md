# Day 6: System Architecture Basics (1h)

- **Topics:** Monolithic vs. microservices; drafting a high-level architecture.  
- **Resources:** Atlassian *“Microservices vs. Monolith”* (defines monolithic and microservice architectures). GeeksforGeeks *“System Design Introduction”* (overview of system design). YouTube: *System Design 101*.  
- **Exercise:** Sketch a high-level design for the “AI Knowledge Assistant” project. Identify components (e.g. frontend UI, FastAPI backend, Vector DB, LLM service) and draw data flow arrows.  
- **Why:** Early architecture planning ensures scalability and maintainability. A monolith is a single deployable unit, whereas microservices split functionality into independent services. Good design covers components and their interactions.
