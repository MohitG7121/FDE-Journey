# Month 1 – Week 4: Finalizing Project 1

- **Goal:** Polish and document the AI Knowledge Assistant for delivery.  
- **Daily Tasks:**  
  - **Day 1:** *Frontend UI (Optional)* – (If desired) Build a simple interface (e.g. Streamlit or React) for the chat assistant. Connect it to the `/ask` API.  
  - **Day 2:** *Containerization* – Write a Dockerfile for the app (install dependencies, copy code). Ensure the app runs in a container.  
  - **Day 3:** *CI/CD Pipeline* – Set up a GitHub Actions workflow or similar: run tests on push, linting, and build Docker image.  
  - **Day 4:** *Deployment* – Deploy the container to a cloud service. (Default: AWS – e.g. Elastic Beanstalk or ECS. Or use Docker Compose locally.)  
  - **Day 5:** *Monitoring & Logging* – Integrate basic logging (to console or a log file). Plan monitoring (e.g. Prometheus + Grafana, or a cloud monitoring tool).  
  - **Day 6:** *Documentation* – Finalize all docs: README, Architecture Doc (with the system diagram), PRD, and start drafting a case study of the project.  
  - **Day 7:** *Review & Interview Prep* – Verify everything works end-to-end. List key learnings and prepare for questions (e.g. “How does RAG improve accuracy?”).  
- **Deliverables:**  
  - Dockerized application, live or mock deployment.  
  - CI/CD config and test suite.  
  - Completed README, PRD, Architecture Doc, and Case Study draft.
